# 随机菜谱主食材配对设计

**背景**

当前页面会从“经典肉类”和“汤品”中随机抽出两道菜，并带有两个现有约束：
- 冬季限定汤品只在指定日期范围内参与抽选
- 如果第一道菜是汤，第二道菜会优先从非汤品中抽取

现在需要新增一条更贴近用餐合理性的规则：同一天的两道菜不应出现明显重复的主食材。用户明确要求：
- `fish + fish` 禁止
- `fish + shellfish` 禁止
- `shellfish + shellfish` 禁止
- `pork / beef / chicken` 组合应更容易出现

**目标**

在不破坏现有页面结构和 Markdown 编辑体验的前提下，为随机逻辑增加“主食材分组 + 配对权重配置”能力，并保留后续人工调参的空间。

## 方案选择

### 方案 A：在随机逻辑中硬编码配对规则

直接在 `drawTwoDishes()` 中写死各类组合的允许与否。

优点：
- 初始改动最小

缺点：
- 规则分散在代码里，可读性差
- 后续调权重必须改逻辑代码
- 注释和维护成本会快速上升

### 方案 B：菜单数据携带 `proteinGroup`，另加配置文件定义组间权重

每道菜只维护一个用于随机的主食材分组字段，随机逻辑读取独立配置文件中的组间权重来决定第二道菜的抽样概率。

优点：
- 数据职责清晰
- 可手动调参，不需要改抽样算法
- 能同时表达“禁止组合”和“低概率组合”
- 适合当前静态页面项目

缺点：
- 需要扩展 Markdown 解析逻辑
- 需要补测试覆盖解析和抽样规则

### 方案 C：把菜单源整体改成 JSON

将 Markdown 列表改为完整对象结构，由 JSON 直接提供所有字段。

优点：
- 结构最明确

缺点：
- 对现有内容编辑方式改动过大
- 当前需求下属于过度设计

## 决策

采用方案 B。

## 数据设计

### 菜单源数据

继续使用 `recipe_menu.md`，但每道菜追加元数据：

```md
1. 红烧肉（百叶结、笋、鸡蛋、油豆腐） | protein=pork
2. 油焖鸡 | protein=chicken
3. 番茄鱼 | protein=fish
4. 虾蟹煲 | protein=shellfish
```

约束：
- 当前只要求一个 `protein` 字段
- 该字段只用于随机搭配，不代表完整配方
- 保留原有菜名文本，不强制改变展示名称

### 配置文件

新增 `src/menuConfig.mjs`，导出：
- `proteinPairWeights`
- `defaultDishWeight`

其中 `proteinPairWeights` 用于描述“第一道菜的主食材组”与“第二道菜的主食材组”的权重关系。权重语义如下：
- `0`：绝对不允许出现该组合
- `> 0`：允许，数值越大越容易被选中

配置文件需要带清晰注释，说明：
- 哪些组合是被用户明确禁止的
- 哪些组合被有意提高概率
- 为什么保留 `defaultDishWeight` 这个暂未启用的扩展点

## 随机逻辑设计

新流程：
1. 读取 Markdown，得到包含 `name`、`category`、`proteinGroup` 的菜对象
2. 按季节规则构造候选列表
3. 先随机抽第一道菜
4. 构造第二道菜候选池：
   - 排除同名菜
   - 如果第一道是汤，优先保留非汤品
5. 对第二道菜候选池计算权重：
   - 查 `proteinPairWeights[first.proteinGroup][candidate.proteinGroup]`
   - 权重为 `0` 的菜不参与抽样
   - 未显式配置时使用默认回退值
6. 按权重抽出第二道菜

如果权重过滤后候选为空，抛出明确错误，提示需要调整配置或补充菜品。

## 错误处理

需要新增两类错误：
- 菜单项缺少 `protein` 标记时报错，避免静默进入错误随机
- 配置导致无可用第二道菜时报错，避免随机结果与规则冲突

## 测试设计

当前项目没有测试基础设施。采用 Node 内建测试器 `node --test`，添加针对 `src/menuLogic.mjs` 的逻辑测试，覆盖：
- Markdown 中 `protein` 字段解析
- 冬季限定规则不被破坏
- `fish + fish`、`fish + shellfish`、`shellfish + shellfish` 不会出现
- `pork / beef / chicken` 组合在可选池中保留且有正权重
- 当第一道为汤时，仍优先搭配非汤品

## 非目标

本次不做：
- 单菜出现频率权重
- 完整配料表建模
- 页面 UI 调参入口
- 将 Markdown 菜谱改造成 JSON 数据源
