# 经典肉类近期选中回避功能设计

## 需求

点击"随机抽今日菜单"按钮抽中某道经典肉类（荤菜）后，该道菜在**接下来的 5 次点击**中不会被再次抽中。

- 只对经典肉类生效
- 仅保存在内存中，页面刷新后重置
- 不影响现有的经典肉类 → 汤品的权重配对逻辑
- 当前经典肉类共有 17 道，排除 5 道后仍有充足候选

## 设计方案：固定大小 FIFO 队列

### 数据结构

在 `src/menuLogic.mjs` 模块级别维护一个队列：

```javascript
const recentlyPickedMeats = [];
const MAX_RECENT_LENGTH = 5;
```

队列存储最近被抽中的经典肉类菜名。

### 执行流程

1. **过滤候选**：当抽取经典肉类时，先从 `section.items` 中过滤掉 `recentlyPickedMeats` 包含的菜
2. **随机抽取**：使用原有的 `pickOne` 从过滤后的候选中抽取
3. **更新队列**：将抽中的菜名推入队列尾部
   - 如果队列长度超过 `MAX_RECENT_LENGTH`，从队列头部移除一个
4. **后续逻辑不变**：汤品仍使用抽中的经典肉类进行权重配对计算，原有逻辑完全保留

### 边界条件

- 页面首次加载：队列为空，不进行过滤，直接从所有菜中抽取
- 队列未满：只过滤已有的菜，剩余全部候选参与抽取
- 题目保证荤菜总数 > 6，所以过滤 5 个后至少还有 2+ 候选，不会出现无候选情况

### 保持现有逻辑

原有设计：
- 经典肉类先抽 → 得到 `classicDish`
- 汤品根据 `classicDish.proteinGroup` 进行权重配对

本设计只在经典肉类抽取前增加过滤步骤，抽取后的流程**完全不变**，不会破坏肉菜-汤品权重逻辑。

## 代码修改范围

- `src/menuLogic.mjs`：增加队列状态，修改 `drawMenuSections` 中经典肉类抽取逻辑
- 其他文件不需要修改：
  - `app.mjs` 不变（UI 不变，点击事件不变）
  - `menuConfig.mjs` 不变（权重配置不变）
  - `index.html` 不变（HTML 结构不变）
  - `styles.css` 不变（样式不变）
